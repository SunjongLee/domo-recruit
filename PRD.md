# DOMO 채용 관리 툴

## 한 줄 요약

지원자에게 고유 URL을 주고, 면접 일정을 스스로 선택하고, 면접관은 라운드별로 평가를 남기며, 지원자는 회사에 직접 질문을 남길 수 있는 채용 관리 툴.

---

## 왜 만드는가 (Why)

- 채용 일정 협의가 지금은 전화·이메일로 이뤄져 HR 담당자의 수동 작업이 많음
- 면접관 평가가 구두로만 공유돼 기록이 남지 않음
- HR 데이터(피드백 등)가 Google Sheets에 저장돼 보안 위험 존재
- 지원자 입장에서 DOMO라는 회사를 채용 과정에서 자연스럽게 알기 어려움

---

## 누구를 위한 것인가 (Who)

| 사용자 | 역할 |
|---|---|
| 관리자 (deuk@domo.co.kr) | 지원자 등록, 상태 관리, 피드백 확인 |
| 지원자 | 면접 일정 선택, 회사 정보 확인, 메시지 작성 |
| 면접관 | 라운드별 평가 항목 작성 |

---

## 무엇을 만드는가 (What)

### 핵심 기능 5가지

1. **지원자 포털** — 고유 URL로 접근, 진행 상태 확인, 면접 슬롯 선택, 회사 소개 열람
2. **지원자 메시지** — 지원자가 회사에 궁금한 점 직접 작성 → 관리자/면접관이 확인
3. **면접관 피드백 (라운드별)** — 1차: 직무역량/커뮤니케이션/업계이해도/태도, 2차: 컬처핏/창의성/브랜드철학/팀어울림
4. **관리자 대시보드** — 지원자 목록, 사이드 패널(메시지·피드백 확인), 지원자 등록
5. **DOMO 회사 소개 섹션** — 지원자 포털에 슬로건·5대 서비스·클라이언트·면접 프로세스 안내

## 기술 스택

| 영역 | 기술 |
|---|---|
| 프레임워크 | Next.js 15 (App Router) |
| DB / Auth | Supabase (PostgreSQL + Auth) |
| 스타일 | Tailwind CSS |
| 코드 관리 | GitHub |
| 배포 | Vercel |
| 외부 연동 | Google OAuth (관리자 로그인), Google Calendar API (면접 슬롯) |

---

## 성공 기준

- 지원자가 URL 하나로 면접 일정을 스스로 선택할 수 있다
- 면접관이 라운드별 평가 링크로 접속해 항목별 평가를 남길 수 있다
- 관리자가 대시보드에서 모든 지원자의 상태·메시지·피드백을 한눈에 볼 수 있다
- HR 데이터가 Supabase DB에 안전하게 저장된다

---

## 하지 않을 것 (1단계 범위 밖)

- 입사 후 온보딩 관리 (2단계)
- 직원 성과 평가 (3단계)
- 자동 이메일 발송 (백엔드 연결 후 추가)
- 채용 공고 페이지 / 외부 지원 폼
- 다중 회사/조직 지원 (DOMO 단독 사용)
